<!-- INTRO -->

**ALEX:** Claude Code 2.1.108 just shipped, and I want to flag three things that actually change how I'll use it this week. The headline is a new environment variable for one-hour prompt cache TTL, and honestly, it's been the biggest bill I didn't realize I was paying.

**SARAH:** Wait, one hour TTL? So you're saying the default five-minute cache was quietly costing everyone money on long sessions? I feel like I need to go set this immediately.

<!-- DISCUSSION -->

**ALEX:** Exactly. Any long refactor or deep debugging session, you hit the five-minute window and the next exchange re-pays for the whole context window. ENABLE_PROMPT_CACHING_1H flips that to an hour across API key, Bedrock, Vertex, and Foundry — all four backends.

**SARAH:** That's a huge deal for anyone billing per token, which is everyone. And there's also a FORCE_PROMPT_CACHING_5M flag as an escape hatch if you actually want the old default?

**ALEX:** Right. If you have some reason to keep the five-minute default — maybe for testing, maybe a compliance thing — you can force it explicitly. And there's a related fix buried in the changelog. If you had DISABLE_TELEMETRY set, you were silently capped at five-minute TTL regardless of your other settings. That's now fixed. So anyone running telemetry-off was basically double-paying without knowing.

**SARAH:** Okay, that's the kind of footgun that costs real money and nobody noticed. What about the startup warning they added for disabled caching?

**ALEX:** Same category. If you have any DISABLE_PROMPT_CACHING variable set — maybe inherited from a shared dotfile or a script from an old project — Claude Code now warns at startup. That's pure visibility. Before, you were silently paying full price every turn with zero cache and no indication anything was off.

**SARAH:** Good. Visibility beats silent defaults every single time. So caching aside, the other thing I keep seeing people react to is this /recap command. What does that actually solve?

**ALEX:** Coming back to a long session mid-week, basically. You had an eight-hour debug thread on Monday, it's Wednesday morning, you run claude in the same project — what do you do? Scroll the transcript? Ask "where were we"? Now you type /recap and it summarizes where the session left off and what was unfinished.

**SARAH:** That is such a workflow change. I was literally doing that manually yesterday. And it's configurable in /config, plus you can even force it on for telemetry-disabled setups with CLAUDE_CODE_ENABLE_AWAY_SUMMARY.

**ALEX:** Right. And paired with the other /resume change — the session picker now defaults to sessions from the current directory instead of listing every session across every project — you get this much cleaner "I'm resuming work in this repo" experience. Ctrl-A widens it back to all projects if you actually need the old behavior.

**SARAH:** That alone saves me two or three scroll-throughs a day. And I saw there's a /undo alias now?

**ALEX:** Yeah, /undo is wired as an alias for /rewind. Muscle memory wins. People kept typing /undo and getting "unknown command," which is just friction for no reason when the team has a perfectly good command with a different name.

**SARAH:** Okay, I have to ask — there's a line in the release notes about the model being able to invoke built-in slash commands through the Skill tool now. /init, /review, /security-review. That sounds architecturally interesting. What actually changed?

**ALEX:** Previously the model couldn't reach its own built-in commands directly. If it was running an agentic task and the right next action was literally /review on a PR or /security-review before a commit, it had to either prompt you or work around the gap. Now those built-ins are exposed through the same Skill tool that runs user-defined skills. The model discovers them and calls them directly.

**SARAH:** So it's closer to a self-service agent now. That's the kind of change that sounds small in a patch note but genuinely shifts the ceiling on what you can automate end-to-end.

**ALEX:** Exactly. It's an integration move more than a new capability, but the compounding effect is real. And the other UX wins in this release are small but they land well. /model now warns you before you switch mid-conversation, because the next response has to re-read the full history uncached against the new model.

**SARAH:** Wait, so people were silently blowing their prompt cache every time they switched models in the middle of a thread?

**ALEX:** Yeah, that's another money footgun surfaced. And error messages got a meaningful cleanup too. Server rate limits are now distinguished from plan usage limits — those are two different problems with two different fixes. 5xx and 529 errors include a direct link to status dot claude dot com. And unknown slash commands suggest the closest match instead of a hard fail.

**SARAH:** So basically if you've been cursing at cryptic Claude Code errors for the past two months, you can stop cursing.

**ALEX:** Stop cursing. Zoom out for a second though — this release is fundamentally a cost-transparency and session-continuity patch. No big new model capability, no new IDE integration, but every feature either saves you money, surfaces a hidden setting, or smooths a daily papercut you'd stopped noticing.
