# Podcast Style Guide — Hard Fork Style

## Overall Tone
A collaborative deep-dive between two smart friends who genuinely enjoy discussing AI. Think Hard Fork (NYT) — informed, accessible, occasionally funny, never condescending. The goal is to make the listener feel like they're eavesdropping on the most interesting conversation at a tech conference dinner.

## Core Principles

1. **Build, don't lecture** — Hosts build on each other's points. Never monologue. Each turn should react to what the other person just said before adding new information.
2. **Yes-and over yes-but** — Default to expanding on the other person's point. Disagreement is welcome but should be productive ("hmm, but have you considered...") not adversarial.
3. **Accessible depth** — Explain complex concepts using analogies. If one host goes deep, the other should translate for the audience.
4. **Genuine reactions** — "Wait, really?", "That's actually huge", "Okay so let me make sure I understand this..." — natural reactions that mirror the listener's experience.
5. **Humor is a spice, not the dish** — Quick wit and occasional jokes, but the substance drives the conversation.

## Episode Structure

### Intro Hook (1-2 turns)
- Host 1 opens with a punchy, attention-grabbing framing of today's topic
- Host 2 reacts with genuine interest/surprise and sets up why this matters

### Topic Introduction (2-3 turns)
- Lay out what happened — the facts
- One host provides context, the other asks clarifying questions
- Establish why the listener should care

### Discussion Beats (4-8 turns depending on length)
- Each beat goes deeper than the last
- Beat pattern: Host 1 makes a point → Host 2 reacts and deepens → Host 1 builds further or pivots to next angle
- Cover multiple angles: technical, business, competitive, societal
- Use phrases that signal depth transitions: "But here's what really gets interesting...", "Okay but zoom out for a second..."

### Forward-Looking Close (2-3 turns)
- "Where does this go from here?"
- Each host offers a prediction or implication
- End on an energizing or thought-provoking note

### Outro (1 turn)
- Brief, warm sign-off
- Tease what's coming or invite the listener to think about something

## Dialogue Patterns

### The "Yes-And" Build
```
HOST_1: [Makes observation]
HOST_2: Right, and what makes that even more interesting is [deeper point]...
```

### The "Hmm But" Pivot
```
HOST_1: [States position]
HOST_2: I see that, but I've been hearing from [customers/researchers] that actually [counter-perspective]...
```

### The Accessible Translation
```
HOST_1: [Technical explanation]
HOST_2: So basically what you're saying is [simpler analogy]?
HOST_1: Exactly, and that means [implication]...
```

### The Genuine Surprise
```
HOST_1: [Drops a surprising fact or number]
HOST_2: Wait, seriously? That's [reaction]. I had no idea it was that [big/fast/different]...
```

## Turn Length
- Each speaker turn should be 2-4 sentences
- Longer turns (4 sentences) for substantive points
- Shorter turns (1-2 sentences) for reactions and transitions
- Never exceed 5 sentences in a single turn — split it

## Words Per Minute Target
- Conversational speaking pace: ~150 words per minute
- 2-minute episode: ~300 words total transcript
- 5-minute episode: ~750 words total transcript
- 10-minute episode: ~1500 words total transcript
